# Getting Started

                                        
The API comes with the following namespaces. You can think of them as categories for functions.

                                        
- [Global Functions](https://gamesense-docs.pages.dev/docs/api/globalFuncs) - API functions that don't belong to any namespaces
- [bit](api/bit.md) - LuaJIT library for bitwise operations
- [client](api/client.md) - General game and cheat-related functions
- [config](api/config.md) - GameSense configuration functions
- [cvar](api/cvar.md) - Utilities for working with game cvars
- [database](api/database.md) - A persistent storage engine that lets you store values between reloads and injects
- [entity](api/entity.md) - Functions for interacting with game entities
- [globals](api/globals.md) - Functions for getting game and cheat-related globals
- [json](api/json.md) - Utilities for working with JSON data, based on lua-cjson
- [materialsystem](api/materialsystem.md) - Functions controlling the games material system, such as modulation, swapping, removing and setting
- [panorama](api/panorama.md) - Functions for interacting with the game engine "Panorama" UI
- [plist](api/plist.md) - Functions for interacting with the player list
- [renderer](api/renderer.md) - A library of functions for drawing and interacting with rendered visuals
- [ui](api/ui.md) - Functions for interfacing with the GameSense user interface
- [vector](api/vector.md) - A built-in vector library
